# 假设检验框架

## 核心概念

假设检验（Hypothesis Testing）是基于样本数据对总体参数做出统计决策的推断方法。

### 原假设与备择假设

| 假设 | 符号 | 含义 |
|------|------|------|
| **原假设（零假设）** | $H_0$ | 无效假设，代表无差异、无效果，通常是"被挑战"的立场 |
| **备择假设（对立假设）** | $H_1$ | 研究假设，代表有差异、有效果 |

**一般原则：** $H_0$ 总是包含等号（$=$, $\leq$, $\geq$），$H_1$ 总是包含不等号（$\neq$, $>$, $<$）。

**示例：**
- 新药与安慰剂疗效比较：$H_0: \mu_{\text{新药}} = \mu_{\text{安慰剂}}$ vs $H_1: \mu_{\text{新药}} \neq \mu_{\text{安慰剂}}$

---

## p值的正确理解

### 正式定义

p值是在**原假设 $H_0$ 为真的前提下**，观察到当前样本结果或更极端结果的概率。

$$
p = P(\text{观测数据} \mid H_0 \text{为真})
$$

### 常见误解

| 错误说法 | 问题 |
|----------|------|
| "p值是H0为真的概率" | p = P(数据\|H0)，不是 P(H0\|数据) |
| "p值越大支持H0的证据越强" | p值大只能说明数据与H0不矛盾，不能"接受"H0 |
| "p=0.04 和 p=0.06 有本质区别" | 不应以0.05为绝对切割，两者信息量相近 |
| "p<0.05 说明效应量很大" | p值受样本量影响，大样本下微小效应也能达显著 |

### 正确解读

- p值只是一个**连续指标**，反映数据与H0的兼容程度
- 不应将p=0.049与p=0.051做二元决策式的区分
- 报告时应给出**精确p值**（如 p=0.037），而非仅 "p<0.05"

---

## 第一类错误与第二类错误

| 决策\真实状态 | $H_0$ 为真 | $H_1$ 为真 |
|--------------|------------|------------|
| **不拒绝 $H_0$** | ✅ 正确决策 ($1-\alpha$) | ❌ 第二类错误 ($\beta$) |
| **拒绝 $H_0$** | ❌ 第一类错误 ($\alpha$) | ✅ 正确决策 ($1-\beta$) |

### 两种错误的关系

- $\alpha \downarrow$ 时 $\beta \uparrow$（样本量固定时）
- 同时降低两者的唯一方法是**增加样本量**
- $\alpha$ 通常设为 0.05 或 0.01
- $\beta$ 通常设为 0.20（对应功效 $1-\beta = 0.80$）

### 统计功效 (Statistical Power)

$$
\text{Power} = 1 - \beta = P(\text{拒绝 } H_0 \mid H_1 \text{ 为真})
$$

功效取决于：$\alpha$、效应量、样本量。

---

## 单侧检验与双侧检验

| 类型 | $H_1$ | 拒绝域 | 适用场景 |
|------|-------|--------|----------|
| 双侧 | $\mu \neq \mu_0$ | 分布两侧 | 无先验方向 |
| 单侧（上） | $\mu > \mu_0$ | 分布右侧 | 有明确方向 |
| 单侧（下） | $\mu < \mu_0$ | 分布左侧 | 有明确方向 |

**原则：** 若无充分的理论依据，应使用**双侧检验**。事后选择单侧检验属于p-hacking。

## Python模拟

### 模拟验证第一类错误

```python
import numpy as np
from scipy import stats

def simulate_type1_error(n=30, n_sim=100000, alpha=0.05):
    """
    模拟：当H0为真时，错误拒绝H0的比例应等于alpha
    从N(0,1)抽样，检验均值是否为0
    """
    reject_count = 0
    for _ in range(n_sim):
        sample = np.random.normal(0, 1, n)
        _, p = stats.ttest_1samp(sample, popmean=0)
        if p < alpha:
            reject_count += 1
    type1_error = reject_count / n_sim
    print(f"模拟第一类错误率: {type1_error:.4f}（理论值: {alpha}）")

simulate_type1_error()
# 输出类似: 模拟第一类错误率: 0.0498（理论值: 0.05）
```

### 模拟验证统计功效

```python
import numpy as np
from scipy import stats

def simulate_power(d=0.5, n=30, n_sim=10000, alpha=0.05):
    """
    模拟：当H1为真时，正确拒绝H0的比例
    从N(0,1)和N(d,1)抽样，检验均值差异
    """
    reject_count = 0
    for _ in range(n_sim):
        group1 = np.random.normal(0, 1, n)
        group2 = np.random.normal(d, 1, n)
        _, p = stats.ttest_ind(group1, group2)
        if p < alpha:
            reject_count += 1
    power = reject_count / n_sim
    print(f"模拟功效（d={d}, n={n}）: {power:.4f}")

# 不同效应量下的功效
for d in [0.2, 0.5, 0.8]:
    simulate_power(d=d)
# 输出类似:
# 模拟功效（d=0.2, n=30）: 0.1456
# 模拟功效（d=0.5, n=30）: 0.4788
# 模拟功效（d=0.8, n=30）: 0.8552
```

### p值分布的可视化

```python
import matplotlib.pyplot as plt
import numpy as np
from scipy import stats

# H0为真时p值的分布（应服从均匀分布）
p_values_h0 = []
for _ in range(10000):
    sample = np.random.normal(0, 1, 30)
    _, p = stats.ttest_1samp(sample, popmean=0)
    p_values_h0.append(p)

# H1为真时p值的分布（左偏）
p_values_h1 = []
for _ in range(10000):
    sample = np.random.normal(0.5, 1, 30)
    _, p = stats.ttest_1samp(sample, popmean=0)
    p_values_h1.append(p)

fig, axes = plt.subplots(1, 2, figsize=(12, 4))
axes[0].hist(p_values_h0, bins=30, density=True, alpha=0.7, edgecolor='black')
axes[0].axhline(y=1.0, color='red', linestyle='--', label='均匀分布')
axes[0].set_title('H0为真时p值的分布')
axes[0].set_xlabel('p值')
axes[0].legend()

axes[1].hist(p_values_h1, bins=30, density=True, alpha=0.7, edgecolor='black')
axes[1].axvline(x=0.05, color='red', linestyle='--', label='α=0.05')
axes[1].set_title('H1为真时p值的分布（d=0.5）')
axes[1].set_xlabel('p值')
axes[1].legend()

plt.tight_layout()
plt.show()
```

## 假设检验七步流程

| 步骤 | 内容 |
|------|------|
| 1 | 建立检验假设（$H_0$ 与 $H_1$） |
| 2 | 确定检验水准 $\alpha$ |
| 3 | 根据研究设计选择检验方法 |
| 4 | 计算检验统计量 |
| 5 | 确定 p 值 |
| 6 | 做出统计推断 |
| 7 | 结合专业知识解释结果 |

::: warning 注意事项
永远不要：
1. 在未看数据前设定 $H_1$ 的方向后，因结果不符合预期而改为另一侧
2. 在p值接近0.05时，"凑"不同的分析方法以获得显著结果（p-hacking）
3. 报告 "p=0.054" 为 "边缘显著" 或 "趋势显著" —— 应当清晰报告结果避免误导
:::
