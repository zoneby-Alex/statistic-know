# 析因设计与裂区设计

## 析因设计 (Factorial Design)

### 原理

析因设计同时考察两个或多个因素，每个因素有多个水平，所有水平组合均被实施。最简形式为 **$2 \times 2$ 析因设计**：两个因素各有两个水平，共 $2^2 = 4$ 个处理组合。

**核心优势：** 可以研究因素间的**交互效应 (Interaction)**——一个因素的效应是否依赖于另一个因素的水平。

**2×2 析因设计示意图：**

```
          因素B
          B1    B2
因素A A1  A1B1  A1B2
     A2  A2B1  A2B2
```

**统计模型：**

$$Y_{ijk} = \mu + \alpha_i + \beta_j + (\alpha\beta)_{ij} + \varepsilon_{ijk}$$

其中 $\alpha_i$ 和 $\beta_j$ 为主效应，$(\alpha\beta)_{ij}$ 为交互效应。

### 交互效应的解读

交互效应存在时，主效应的意义需要谨慎解释。例如：药物 A 和药物 B 联合使用时，若存在拮抗交互（交互效应为负），单独看 A 有效、B 有效，但 A+B 未必更好。

```python
import numpy as np
import pandas as pd
import statsmodels.api as sm

# 模拟 2×2 析因设计数据
np.random.seed(42)
n_per_group = 10
data = []
for drug_a in [0, 1]:
    for drug_b in [0, 1]:
        # 假设有交互效应：A 和 B 联合时效果大于相加
        effect = 10 + 3*drug_a + 3*drug_b + 2*drug_a*drug_b
        y = effect + np.random.normal(0, 2, n_per_group)
        for val in y:
            data.append({'A': drug_a, 'B': drug_b, 'response': val})

df = pd.DataFrame(data)

# 拟合含交互项的模型
model = sm.OLS.from_formula('response ~ C(A) * C(B)', data=df)
result = model.fit()
print(result.summary())
```

### 全因子设计的效率

相比于分别做两个单因素实验，析因设计在同样样本量下可以获得更多信息——不仅能检验两个主效应，还能检验交互效应。**全因子设计是检验交互效应的唯一途径。**

## 裂区设计 (Split-Plot Design)

### 原理

裂区设计产生于当某些因素难以随机化或改变水平成本较高时。将实验单位分为"主区"（whole-plot），在主区内再分为"子区"（sub-plot）。**主区因素**作用于整个主区，**子区因素**作用于主区内的子区。

典型的应用场景：农业田间试验中，灌溉方式（主区因素）难以频繁改变，而品种（子区因素）可以随机分配。

**统计模型：**

$$Y_{ijk} = \mu + \alpha_i + \beta_{j(i)} + \gamma_k + (\alpha\gamma)_{ik} + \varepsilon_{ijk}$$

其中 $\alpha_i$ 为主区处理效应，$\beta_{j(i)}$ 为主区误差（主区内的区组变异），$\gamma_k$ 为子区处理效应，$(\alpha\gamma)_{ik}$ 为交互效应。

### 不同精度的含义

- **主区因素**：以主区为重复单位，样本量 = 主区数，检验功效较低
- **子区因素**：以子区为重复单位，样本量 = 子区总数，检验功效较高
- **交互效应**：通常以子区误差检验，功效也较高

### 适用场景

| 场景 | 主区因素 | 子区因素 | 原因 |
|------|---------|---------|------|
| 田间试验 | 灌溉/耕作方式 | 品种 | 灌溉大区实现 |
| 工业实验 | 温度/压力 | 原料批次 | 温度难调 |
| 临床试验 | 医院/中心 | 治疗方案 | 中心随机化困难 |

### Python 模拟裂区设计分析

```python
import statsmodels.api as sm

# 模拟裂区设计
# 主区因素: irrigation (I1, I2)
# 子区因素: variety (V1, V2, V3)
# 主区单位: block内嵌套的主区

data = pd.DataFrame({
    'yield': [...],        # 观测值
    'irrigation': [...],   # 主区因素
    'variety': [...],      # 子区因素
    'block': [...],        # 区组
    'mainplot': [...]      # 主区ID（嵌套于block内）
})

# 裂区设计的ANOVA模型：主区误差项需指定
model = sm.OLS.from_formula(
    'yield ~ C(block) + C(irrigation) + C(variety) '
    '+ C(irrigation):C(variety)',
    data=data
)
result = model.fit()
```

## 与 CRD / RBD 的对比

| 特性 | CRD | RBD | 析因设计 | 裂区设计 |
|------|-----|-----|---------|---------|
| 控制混杂 | 无 | 一个因素 | 可分析交互 | 不同精度 |
| 交互效应 | 不可检 | 不可检 | **可检** | **可检** |
| 随机化层次 | 单层 | 单层 | 单层 | **两层** |
| 分析复杂度 | 低 | 低 | 中 | 高 |
| 子区检验功效 | — | — | — | **较高** |

> **要点总结：** 析因设计的最核心价值是检验交互效应；裂区设计的核心特征是存在两个层次的随机化单元，主区因素的检验功效低于子区因素。在设计时如需研究因素间的协同或拮抗作用，应优先选择析因设计。
