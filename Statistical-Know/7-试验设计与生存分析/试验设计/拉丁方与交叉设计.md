# 拉丁方设计与交叉设计

## 拉丁方设计 (Latin Square Design)

### 原理

拉丁方设计可控制**两个**已知混杂因素，是随机区组设计的扩展。将两个区组因素分别安排为"行"和"列"，处理因素以拉丁方阵形式排列，使每个处理在每行每列各出现一次。

**3×3 拉丁方示例：**

```
       列1  列2  列3
行1:    A    B    C
行2:    B    C    A
行3:    C    A    B
```

**统计模型：**

$$Y_{ijk} = \mu + \alpha_i + \beta_j + \tau_k + \varepsilon_{ijk}$$

其中 $\alpha_i$ 为行效应，$\beta_j$ 为列效应，$\tau_k$ 为处理效应。

### 适用场景

- **农业试验**：控制土壤肥力的两个方向梯度
- **工业实验**：控制机器和操作员两个因素
- **动物实验**：控制窝别和体重两个因素

### 优缺点

| 优点 | 缺点 |
|------|------|
| 同时控制两个混杂因素 | 处理数必须等于行/列数（至少 $k \geq 4$ 才有足够误差自由度） |
| 减少实验单位数量 | 可加性假设较强（行×列、行×处理等交互效应不存在） |
| 统计效率高于 CRD 和 RBD | 数据缺失时分析复杂 |

## 交叉设计 (Crossover Design)

### 原理

同一受试对象先后接受不同的处理，每个受试者充当自己的对照。经典的 **2×2 交叉设计** 将受试者随机分为两组：一组先接受处理 A 后接受处理 B（序列 AB），另一组为先 B 后 A（序列 BA）。

**2×2 交叉设计示意图：**

```
序列AB:  [阶段1] 处理A  →  洗脱期  →  [阶段2] 处理B
序列BA:  [阶段1] 处理B  →  洗脱期  →  [阶段2] 处理A
```

**统计模型：**

$$Y_{ijk} = \mu + \pi_j + \tau_{(i,k)} + \gamma_i + \varepsilon_{ijk}$$

其中 $\pi_j$ 为阶段效应，$\tau_{(i,k)}$ 为处理效应，$\gamma_i$ 为序列效应（即个体效应）。

### 关键概念

**自身对照：** 每名受试者既是试验组又是对照组，消除了个体间变异。

**序列效应 (Carryover Effect)：** 第一阶段处理对第二阶段的影响。若有序列效应，交叉设计的结果将被污染。这是交叉设计最大的威胁。

**洗脱期 (Washout Period)：** 两个阶段之间的间隔，目的就是让前一阶段的药物完全代谢消除。洗脱期长度通常 $\geq 5$ 个药物半衰期。

### 适用场景

- 慢性病（高血压、哮喘、糖尿病）的等效性/非劣效性试验
- 生物等效性研究（BE研究）
- 不适于急性病或可治愈疾病

### 优缺点

| 优点 | 缺点 |
|------|------|
| 样本量需求小（约为平行组的 1/2） | 可能存在序列效应 |
| 个体变异完全控制 | 不适于急性病或疗程过长的疾病 |
| 统计检验效率高 | 受试者退出率高（时间长） |
| 每个受试者都接受所有治疗 | 复杂的设计（多阶段、多序列） |

## 统计分析要点

### 拉丁方分析

```python
import statsmodels.api as sm
import pandas as pd

# 假设数据：yield, row, col, treatment
df = pd.DataFrame({
    'yield': [23, 28, 21, 25, 30, 22, 20, 24, 27],
    'row': list('AAABBBCCC'),
    'col': list('123123123'),
    'treat': list('ABCBCACAB')
})

# 拟合拉丁方模型
model = sm.OLS.from_formula('yield ~ C(row) + C(col) + C(treat)', data=df)
result = model.fit()
print(result.summary())
```
