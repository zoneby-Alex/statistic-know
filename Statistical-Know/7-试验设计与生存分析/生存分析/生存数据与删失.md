# 生存数据与删失

## 时间到事件数据 (Time-to-Event Data)

生存分析关注的是从某个起始时间点到某"事件"发生所经过的时间。这类数据称为**时间到事件数据** (time-to-event data) 或 **生存时间数据**。

**常见事件定义：** 死亡、疾病复发、肿瘤进展、设备失效、患者出院等。

**生存数据的两个关键特征：**
1. 非负性：生存时间 $T \geq 0$
2. **删失 (Censoring)**：部分研究对象在观察期内未发生事件

---

## 三种删失类型

### 右删失 (Right Censoring) —— 最常见

研究对象在观察期结束时仍未发生事件，或中途失访。只知道该对象的生存时间**大于**某个值 $C$。

$$Y_i = \min(T_i, C_i), \quad \delta_i = I(T_i \leq C_i)$$

其中 $Y_i$ 为观察时间，$T_i$ 为真实生存时间，$C_i$ 为删失时间，$\delta_i$ 为事件指示（1=事件发生，0=删失）。

**常见原因：**
- 研究结束时仍未发生事件
- 失访（患者搬家、拒绝随访）
- 竞争风险（死于其他原因）

### 左删失 (Left Censoring)

事件在观察开始前已经发生，但确切时间未知。在生物医学研究中相对少见。

**例：** 检测某生物标志物达到阈值的时间，在第一次检测时已超过阈值。

### 区间删失 (Interval Censoring)

只知道事件发生在某个时间区间 $(L, R]$ 内，但确切时间未知。常见于定期随访的研究。

**例：** 患者每 3 个月复查一次，在两次复查之间发生疾病进展，我们只知道进展时间在 $(t_{k-1}, t_k]$ 之间。

---

## 生存函数与风险函数

### 生存函数 $S(t)$

$$S(t) = P(T > t) = 1 - F(t)$$

表示生存时间超过 $t$ 的概率。$S(0) = 1$，随时间增加单调递减。

### 风险函数 $h(t)$

$$h(t) = \lim_{\Delta t \to 0} \frac{P(t \leq T < t + \Delta t \mid T \geq t)}{\Delta t}$$

表示在已存活到 $t$ 时刻的条件下，瞬间发生事件的概率密度。$h(t)$ 可以是非单调的（如浴盆曲线）。

**两者的数学关系：**

$$S(t) = \exp\left(-\int_0^t h(u) du\right)$$

$$h(t) = -\frac{d}{dt} \log S(t)$$

---

## 删失不是缺失值

这是生存分析中最常见的误解。**删失提供了有价值的部分信息**——我们知道该对象至少存活到了某个时间点。

- **缺失值**：完全不知道观测值
- **右删失**：知道 "$T > C$"，提供了下界信息

如果删失与事件发生独立（**非信息性删失**），则分析中可以正确利用删失数据。若删失与事件相关（如因不良反应退出研究），则可能导致估计偏倚。

---

## Python 生成生存数据

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

np.random.seed(42)

def generate_survival_data(n=200, lambda_event=0.1, lambda_censor=0.05):
    """生成包含右删失的生存数据"""
    # 真实生存时间 ~ 指数分布
    true_times = np.random.exponential(1/lambda_event, n)
    # 删失时间 ~ 指数分布
    censoring_times = np.random.exponential(1/lambda_censor, n)
    # 观察时间 = min(真实时间, 删失时间)
    observed_times = np.minimum(true_times, censoring_times)
    # 事件指示
    event_occurred = (true_times <= censoring_times).astype(int)

    return pd.DataFrame({
        'subject_id': range(1, n+1),
        'time': observed_times,
        'event': event_occurred
    })

df = generate_survival_data(50)
print(df.head(10))
print(f'\n删失率: {1 - df["event"].mean():.1%}')

# 观察删失分布
plt.hist(df[df['event']==1]['time'], alpha=0.7, label='事件发生', bins=15)
plt.hist(df[df['event']==0]['time'], alpha=0.7, label='删失', bins=15)
plt.xlabel('观察时间')
plt.ylabel('频数')
plt.legend()
plt.title('生存数据：事件 vs 删失')
plt.show()
```

> **要点总结：** 删失是生存数据的内在特征而不是数据质量问题。右删失最为常见，其核心假设是非信息性删失。理解 $S(t)$ 和 $h(t)$ 两个函数是理解后续 KM 曲线和 Cox 模型的基础。
